# Digismartz Technologies — Website **Transforming Ideas into Digital Success.** A fast, modern, mobile-responsive static website for Digismartz Technologies — a full-service digital marketing and branding company. Built with plain HTML, CSS, and JavaScript (no build step required), ready to deploy on **GitHub Pages** or **Cloudflare Pages**. ## Pages | Page | File | |---|---| | Home | `index.html` | | About Us | `about.html` | | Services (all 14) | `services.html` | | Portfolio (with filters) | `portfolio.html` | | Contact + Free Consultation form | `contact.html` | ## Features - Black + Electric Blue premium dark theme, mobile responsive - Floating **WhatsApp** and **Call** buttons on every page - Free consultation form, contact info, newsletter signup, FAQ accordion - Portfolio category filters, scroll-reveal animations, animated "signal" motif - SEO meta tags, `robots.txt`, `sitemap.xml` - No frameworks, no build step — loads fast, Core Web Vitals friendly ## Deploy — Option A: Cloudflare Pages (preferred) 1. Push this folder to a GitHub repository (steps below). 2. In the [Cloudflare dashboard](https://dash.cloudflare.com) → **Workers & Pages → Create → Pages → Connect to Git**. 3. Select the repository. 4. Build settings: - **Framework preset:** None - **Build command:** *(leave empty)* - **Build output directory:** `/` 5. Click **Save and Deploy**. Your site goes live at `https://.pages.dev` with SSL enabled automatically. 6. When you buy a domain, add it under **Custom domains** in the Pages project. ## Deploy — Option B: GitHub Pages 1. Push to GitHub (below). 2. Repo → **Settings → Pages → Source: Deploy from a branch** → branch `main`, folder `/ (root)` → Save. 3. Site goes live at `https://.github.io//`. ## Push to GitHub ```bash cd digismartz-website git init git add . git commit -m "Initial website for Digismartz Technologies" git branch -M main git remote add origin https://github.com//.git git push -u origin main ``` ## Before launch — TODO - [ ] **Forms:** The contact/consultation and newsletter forms currently show a confirmation message only.
